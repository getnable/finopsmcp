Based on everything we found this session, here's the specific playbook:

Step 1: Find out what you're actually calling (5 minutes)

Run this in your AWS account for each operation to see where the volume is:

aws cloudwatch get-metric-statistics \
  --namespace AWS/Textract \
  --metric-name SuccessfulRequestCount \
  --dimensions Name=Operation,Value=AnalyzeDocument \
  --start-time 2026-04-27T00:00:00Z \
  --end-time 2026-05-27T00:00:00Z \
  --period 2592000 --statistics Sum

# Run again for each:
# Value=DetectDocumentText
# Value=AnalyzeExpense
# Value=AnalyzeID

This tells you which API you're hammering and the exact volume. Everything else follows from this.

Step 2: Kill non-prod Textract (highest confidence fix)

Your log groups show 4 environments running Textract Lambdas:

  • company-textract-lambda-prd
  • company-textract-lambda-qa
  • company-textract-lambda-stg
  • company-textract-lambda-dev (inferred)

In your Lambda code, add an environment gate:

import os

def process_document(document):
    if os.environ.get("ENV") == "production":
        return call_textract(document)
    else:
        return load_fixture_response(document)  # pre-saved Textract output

Save a handful of real Textract responses as JSON fixtures for QA/staging to use. Non-prod never touches the API again.

Step 3: Check if you're using the right API tier

Pricing per 1,000 pages:

  • DetectDocumentText: $1.50 (raw text only)
  • AnalyzeDocument tables: $15
  • AnalyzeDocument forms: $50
  • AnalyzeExpense: $10

If your downstream code only needs the raw text blocks, switch from AnalyzeDocument to DetectDocumentText. One line change, 10-33x cheaper.

# Before (expensive if you only need text)
response = textract.analyze_document(
    Document={"S3Object": {"Bucket": bucket, "Name": key}},
    FeatureTypes=["TABLES", "FORMS"]
)

# After (if you only use Blocks of type LINE or WORD)
response = textract.detect_document_text(
    Document={"S3Object": {"Bucket": bucket, "Name": key}}
)

Step 4: Add a result cache

If the same documents get reprocessed (retries, re-runs, duplicate uploads), cache the output:

import hashlib, json, boto3

def get_textract_result(bucket, key):
    doc_hash = hashlib.md5(f"{bucket}/{key}".encode()).hexdigest()
    cache_key = f"textract-cache/{doc_hash}.json"

    s3 = boto3.client("s3")

    # Check cache first
    try:
        cached = s3.get_object(Bucket=bucket, Key=cache_key)
        return json.loads(cached["Body"].read())
    except s3.exceptions.NoSuchKey:
        pass

    # Cache miss, call Textract
    result = textract.detect_document_text(
        Document={"S3Object": {"Bucket": bucket, "Name": key}}
    )
    s3.put_object(Bucket=bucket, Key=cache_key, Body=json.dumps(result))
    return result

S3 storage for cached results is essentially free compared to Textract API costs.

Order of operations

  1. Run the CloudWatch query first, know your baseline
  2. Ship the non-prod environment gate, fastest savings, lowest risk
  3. Evaluate API tier, only if Step 1 shows you're using AnalyzeDocument
  4. Add caching, if Step 1 shows repeated document processing
Step 2 alone could save $960–$1,900/mo with a deploy, no accuracy risk, no behavioral change in prod.
